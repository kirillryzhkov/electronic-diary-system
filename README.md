# Electronic Diary System

**Electronic Diary System** — дипломный проект информационной системы электронного дневника на Django.

Проект предназначен для управления учебными группами, предметами, кабинетами, расписанием, посещаемостью, домашними заданиями и оценками студентов.

Система поддерживает роли пользователей, REST API, JWT-авторизацию, Swagger-документацию, мультиязычный интерфейс и светлую/тёмную тему.

## Роли пользователей

### Администратор

Администратор управляет учебной структурой системы:

- создаёт и редактирует группы;
- создаёт и редактирует кабинеты;
- создаёт и редактирует предметы;
- назначает преподавателей на группы и предметы;
- управляет расписанием занятий;
- видит всю посещаемость;
- видит все домашние задания;
- видит все оценки;
- имеет доступ к административной панели Django.

### Преподаватель

Преподаватель работает только со своими назначениями:

- видит только свои группы;
- видит только свои предметы;
- видит своё расписание;
- отмечает посещаемость только по своим группам;
- добавляет домашние задания только для своих групп;
- выставляет оценки студентам из своих групп;
- видит оценки, которые поставил сам;
- может редактировать и удалять только свои записи;
- может просматривать статистику.

### Студент

Студент имеет ограниченный доступ:

- видит только свои оценки;
- видит расписание своей группы;
- видит свою посещаемость;
- видит домашние задания своей группы;
- видит комментарии преподавателя к оценкам;
- видит дату и время выставления оценки.

## Установка проекта

### 1. Клонировать репозиторий

```bash
git clone https://github.com/kirillryzhkov/electronic-diary-system.git
cd electronic-diary-system
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv
```

### 3. Активировать виртуальное окружение

Для Windows:

```bash
.venv\Scripts\activate
```

### 4. Установить зависимости

```bash
pip install -r requirements.txt
```

### 5. Выполнить миграции

```bash
python manage.py migrate
```

### 6. Создать демо-данные

```bash
python manage.py seed_demo
```

### 7. Запустить сервер

```bash
python manage.py runserver
```

После запуска проект будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

## Авторизация API

Для получения JWT-токена нужно отправить POST-запрос:

```text
POST /api/v1/token/
```

Пример тела запроса:

```json
{
  "username": "teacher",
  "password": "password123"
}
```

Ответ содержит `access` и `refresh` токены:

```json
{
  "refresh": "...",
  "access": "..."
}
```

Для авторизации в Swagger нужно нажать кнопку **Authorize** и вставить:

```text
Bearer <access_token>
```

Проект поддерживает три языка интерфейса:

```bash

python manage.py compilemessages
```
