# Electronic Diary System

**Electronic Diary System** — дипломный проект информационной системы электронного дневника, разработанный на Django.

Проект предназначен для учёта учебных групп, предметов, кабинетов, назначений преподавателей и оценок студентов.  
Система поддерживает разделение ролей, REST API, JWT-авторизацию, Swagger-документацию, мультиязычный интерфейс и светлую/тёмную тему.

---

## Основные возможности

- Авторизация пользователей
- Разделение ролей:
  - Администратор
  - Преподаватель
  - Студент
- Управление пользователями
- Управление учебными группами
- Управление предметами
- Управление кабинетами
- Назначение преподавателей на предметы и группы
- Выставление оценок студентам
- Пояснение к каждой оценке
- Отображение даты и времени выставления оценки
- Просмотр оценок по ролям
- Страница “Мои группы” для преподавателя
- Расчёт среднего балла
- Статистика по предметам
- Светлая и тёмная тема
- Мультиязычный интерфейс:
  - Русский
  - Английский
  - Казахский
- REST API
- Swagger-документация
- Автоматические тесты
- Демо-данные одной командой

## Роли пользователей

### Администратор

Администратор управляет учебной структурой системы:

- создаёт и редактирует группы;
- создаёт и редактирует кабинеты;
- создаёт и редактирует предметы;
- назначает преподавателей на группы и предметы;
- видит все оценки в системе;
- имеет доступ к административной панели Django.

### Преподаватель

Преподаватель работает только со своими назначениями:

- видит только свои группы;
- видит только свои предметы;
- видит только оценки, которые поставил сам;
- может добавлять оценки студентам из своих групп;
- может редактировать и удалять только свои оценки;
- может просматривать статистику.

### Студент

Студент имеет ограниченный доступ:

- видит только свои оценки;
- видит предметы;
- видит комментарии преподавателя к оценкам;
- видит дату и время выставления оценки.
